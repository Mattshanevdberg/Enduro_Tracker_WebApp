## Validated dashboard structure

### 1. Hero section

The upper part of the page will use a full-width race image as the background.

The displayed image changes when the selected tab changes:

| Tab            | Suggested image                        |
| -------------- | -------------------------------------- |
| Upcoming Races | Riders waiting at the start line       |
| Live Races     | Fast-moving riders with motion blur    |
| Past Races     | Riders during sunset or after an event |
| Riders         | Group portrait of riders               |

A dark gradient overlay will sit over the image so that the navigation and text remain readable.

### Header layout

**Top left**

* Logo placeholder
* `Kooksnylive` wordmark

**Top right**

Two icon-and-text controls:

* Motorbike rider illustration above `Login`
* Rider standing beside a bike above `Sign Up`

For the mock-up, these will use simple illustration placeholders.

### Hero text

The hero text changes with the selected tab.

Example:

```text
Upcoming Races

Find your next race, prepare your equipment and secure your tracker.
```

Each tab will have its own heading and short welcome message.

### Tab navigation

A rounded navigation bar will sit near the lower centre of the hero section:

```text
Upcoming Races | Live Races | Past Races | Riders
```

* `Upcoming Races` is selected by default.
* The selected tab uses the green accent colour.
* Clicking a tab changes:

  * the hero image;
  * the hero heading;
  * the hero description;
  * the list content below.

This will be handled with simple JavaScript in the mock-up.

---

## 2. Scrolling behaviour

On desktop:

* the hero and tab navigation remain visible;
* the race or rider list scrolls below them.

On smaller screens:

* the hero will be reduced in height;
* the page will use normal vertical scrolling so the fixed section does not occupy most of the phone screen.

---

## 3. Race-list layout

The race section will use a flat list style similar to the Red Bull event example.

Each race row will contain:

### Left

* Race logo placeholder

### Centre

* Optional status badge
* Race name
* Variable-length description
* Location with a small location icon

### Right

* Large bold date
* Relevant action button

Example desktop structure:

```text
[Race logo]   Race name and details                  12 AUG
              Description                            [Get Device]
              Location
```

### Row behaviour

The complete row will be clickable and will open the race page.

The action button will operate separately and will not trigger the row link.

### Buttons by tab

| Tab            | Button     |
| -------------- | ---------- |
| Upcoming Races | Get Device |
| Live Races     | No button  |
| Past Races     | Results    |

For live races, the date area can include a noticeable green `LIVE` status indicator.

---

## 4. Rider-list layout

Each rider row will contain:

### Left and centre

* Rider name
* Team
* Bike
* Short biography

### Right

* Rider profile-image placeholder
* `View Rider` button below the image

Example:

```text
Rider name                                     [Profile image]
Team                                           [View Rider]
Bike
Short biography
```

The row and the button can both open the rider profile.

---

## 5. Visual style

### Colour palette

* Near-black background and headings
* White content area
* Light grey page background
* Medium grey supporting text
* Green accent for:

  * selected tabs;
  * primary buttons;
  * live indicators;
  * hover states.

Suggested accent:

```css
#19a463
```

### Components

* Rounded navigation bar
* Rounded buttons
* Rounded race-logo and rider-image placeholders
* Soft shadows
* Thin separators between rows
* Slight row elevation or background change on hover
* Bold, modern headings
* Generous spacing without making the list feel oversized

---

## 6. Responsive layout

### Desktop

* Wide horizontal list rows
* Logo on the left
* Details in the centre
* Date and action on the right

### Mobile

Each race becomes a compact stacked card:

```text
[Race logo]

Race name
Description
Location

Date                  [Action]
```

For riders:

```text
[Profile image]

Rider name
Team
Bike
Biography

[View Rider]
```
